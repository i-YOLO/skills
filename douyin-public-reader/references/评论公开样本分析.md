# 评论公开样本分析

## 定义

“公开评论样本”指未登录公开页自然展示出来的少量评论文本，以及在不触发登录的前提下，通过有限点击“展开回复”或“点击加载更多”看到的少量新增评论 / 回复文本。它不是完整评论区，不是热门评论完整榜，也不是粉丝画像。

如果页面显示“请先登录后发表评论”“登录后即可参与互动讨论”，这本身是未登录信号。可以继续记录页面已经展示的公开评论样本，也可以按限制有限点击公开可见的“展开回复”和“点击加载更多”；但不得登录、不得输入任何账号信息、不得绕过限制。

## 可以记录的字段

- 评论文本。
- 公开可见的时间和地区。
- 公开可见的点赞数。
- 作者公开回复。
- 评论指向的关注点。

## 不可以做的动作

- 不点击“立即登录”。
- 不为了评论输入手机号、验证码、密码或扫码。
- 不批量翻页评论。
- 不高频点击、无限滚动或追求完整评论区。
- 不在触发登录弹窗、验证码、扫码、手机号输入或页面阻断后继续尝试。
- 不把“点击加载更多”作为稳定可用能力；每条视频最多尝试 3 次。
- 不把公开样本称为完整热门评论。
- 不把公开样本称为完整粉丝群体画像。

## 有限展开规则

- “展开回复”：确认未登录公开视角后可以低频点击，用于读取少量二级回复样本。
- “点击加载更多”：确认未登录公开视角后，每条视频最多点击 3 次。
- 每次点击后必须重新检查页面状态；如果出现登录弹窗、验证码、扫码、手机号输入、页面阻断、按钮不可点击或无新增内容，停止该动作并记录限制。
- 只保留对选题、账号拆解或用户关注点判断有价值的样本，不追求全量。

## 分析方法

先按单条视频归纳评论关注点，再跨视频合并主题。

常见关注点包括：

- 安装、注册、验证和安全使用。
- 模板、提示词、文件、教程、工作流。
- 成本、资源、配置、算力和 token 消耗。
- 工具兼容性、代理、替代工具和排障。
- 真实变现、接单可信度和交付路径。
- 交付质量、效果翻车和失败复盘。
- 项目上下文、文档模板和工程化方法。

## 输出口径

必须写清：

- 样本来自未登录公开页自然展示区域。
- 如有展开，说明样本包含少量有限展开回复或加载更多后的公开样本。
- 样本数量有限。
- “展开回复”的点击次数。
- “点击加载更多”的点击次数，且每条视频不得超过 3 次。
- 没有读取完整评论区。
- 结论是“基于公开样本推断”，不是完整粉丝画像。

推荐句式：

```
以下评论仅为未登录公开页自然展示的少量样本，以及不触发登录的有限展开样本，可以用于选题判断，但不能代表完整评论区或完整热门评论榜。
```

```
粉丝关注点为基于公开评论样本的推断，不等同于完整用户画像。
```
