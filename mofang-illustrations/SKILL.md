# Mofang Illustrations

## 核心定位

把中文知识内容翻译成可交付的原创视觉资产，并用内置 `image_gen` 完成真实生成、编辑、评图和位图交付。

默认核心风格是“手绘系统草图风”：墨方正在搭系统的原创视觉隐喻。墨方是一只章鱼型系统架构师：圆头、圆框眼镜、半眯眼、黑色领结、多条承担实际工作的触手，常操作系统魔方。让墨方始终承担画面的核心动作，不要把它当吉祥物或装饰。

本 skill 也支持“3D 材质解释图风”，用于流程图、机制图、图表美化、教育解释和带标签中心图。对用户展示风格时，只描述风格，不使用“墨方”或“归藏”作为选项名称；内部风格 ID 见 `references/style-registry.md`。

## 三条路线

## 硬性 Preflight 闸门

只要用户说“做配图”“生成配图”“正文配图”“解释图”“图表美化”“批量配图”“做 N 张图”等，默认视为真实生图意图，必须先发出完整 preflight 确认面板。

除非用户明确说“只要 shot list / brief / prompt”“不要生图”“不调用 Image Gen”，否则不要直接进入策略模式。

在完成必要 preflight 之前，不要：

- 创建输出目录。
- 写入 `brief/`、`prompts.md` 或 `run-manifest.md`。
- 承诺已经确定最终风格、标签或执行策略。
- 调用 `image_gen` 或安排子 Agent 生图。

如果用户选择“正式落盘”，也不要使用写死目录；必须在交付形式问题里同时确认使用推荐目录或由用户提供目录，之后才允许创建文件。

如果用户说“测试流程”，也要先发出完整 preflight 确认面板，用来测试确认流程；只有当用户选择“暂不生图”或“只要 prompt”时，才停止在文本阶段。

### 1. 策略模式

仅当用户明确只要策略、shot list、brief 或 prompt，或在 preflight 中选择“暂不生图 / 只要 prompt”时：

1. 提炼核心观点、认知转折和视觉锚点。
2. 短内容输出 1–3 张、常规内容输出 4–8 张 shot list。
3. 为每张图写清放置位置、核心意思、结构类型、墨方动作、保留项、可改项、禁止项、比例和交付形式。
4. 不调用 `image_gen`，不声称已经生成图片。

### 2. 生成模式

当用户要求真实生图时：

1. 先按 `references/workflow.md` 发出完整 preflight 确认面板，再确定单图 brief 或完整 shot list。
2. 生图前必须一次性确认：视觉风格、是否先测试图、多图执行方式、图内文字、交付形式与保存位置。
3. 每个问题都提供推荐选项；用户回复“默认/推荐”时，采用全部推荐项。用户只回答部分选项时，未回答项采用推荐值，但正式落盘目录和自定义标签除外。
4. 已能从上下文推断的问题可以跳过，但在调用 `image_gen` 前必须用一句话说明采用的默认值。
5. 将任务归类为 `stylized-concept`，使用 `references/style-registry.md` 选择风格，再用 `references/prompt-template.md` 组织 prompt。
6. 手绘系统草图风默认使用 `assets/avatar/01-mofang-social-avatar.png` 作为角色身份参考图；仅在世界观设定图任务中追加 `assets/avatar/02-mofang-worldview-ip-sheet.png`。
7. 3D 材质解释图风默认不使用墨方头像参考图，除非用户明确要求把墨方也放入该风格。
8. 默认调用内置 `image_gen`；多图任务保持一张图片一次调用，不用一次请求代替多个资产，不因为“批量”切换 CLI。
9. 等当前批次全部结束后统一检查输出；将不合格资产组成返修批次，每张只改变一个变量。
10. 预览任务内联展示；正式任务按交付规则落盘并报告绝对路径、最终 prompt、执行策略、风格和所用模式。

### 3. 编辑模式

当用户要求修改现有图片时：

1. 明确标注每张输入图的角色：编辑目标、角色参考、风格参考或合成素材。
2. 本地编辑目标尚未出现在对话中时，先用 `view_image` 检查。
3. 列出“只改什么”和“必须保持什么”，并把不变项写进每轮 prompt。
4. 使用内置 `image_gen` 编辑，默认另存版本化文件，不覆盖原图。
5. 对身份、构图、文字和未指定区域逐项复核。

## 多图执行策略

- 平衡模式 / `balanced`：推荐，适合 3–8 张。先生成 1 张无复杂文字、最能代表角色和风格的测试/锚点图；通过 QA 后，以最大并发数 3 生成其余资产。
- 快速模式 / `fast`：适合 8 张以上或快速探索。可使用多个子 Agent 并行生成独立资产；主 Agent 必须制定统一 brief、asset_id、风格规则和 QA 标准，并负责最终收口。
- 稳定模式 / `strict`：适合 1–2 张或重要成品。逐张生成并检查。单图任务跳过多图执行问题，默认使用稳定模式。
- 同一编辑目标的连续修改必须串行；不同目标的独立编辑可以按所选策略并发。
- 单张失败不得取消其他资产。只重试失败资产一次，并把重试批次的最大并发数降为 2；再次失败时记录错误并继续交付其他成功资产。

## Image Gen 执行规则

- 正文配图默认 16:9 横版；头像默认 1:1；世界观设定图默认 1:1，除非用户指定。
- 默认生成无文字图片；但解释图、流程图、机制图、图表美化、教育解释等场景，必须先给出 3–5 个短标签候选并询问用户是否使用。
- 用户选择使用推荐标签时，直接使用候选标签；用户选择修改标签但未给出标签时，不调用生图，继续追问标签。
- 图内标签每个优先 2–5 个中文字符，最多 8 个；必须逐字写入 prompt 并专项核对准确性。
- 图表输入必须先抽取图表类型、标题、类别顺序、坐标、单位、数值和误差线。无法读准数值时必须询问或标注不确定，不得编造。
- 冷门、科学、历史、品牌、模型、装置或具体地点对象，必须先查参考事实和稳定视觉线索，再写 prompt。
- 参考图只锁定角色身份和风格 DNA，不照搬其具体构图、文案或节点布局。
- 并发只改变调用编排，不改变 prompt、参考图、QA、单变量迭代或文件覆盖保护。
- 内置 `image_gen` 不可用时，说明 CLI fallback 需要 `OPENAI_API_KEY`；只有用户明确同意后才切换。
- 简单透明背景请求先生成纯色色键底图，再使用系统 `imagegen` Skill 的 `remove_chroma_key.py` 去背并验证 alpha。
- 遇到毛发、烟雾、玻璃、液体、半透明或反光主体时，先说明原生透明需要 CLI `gpt-image-1.5`，取得用户确认后再执行。
- 不创建或维护自定义生图 API 脚本。

## 参考图角色

- `assets/avatar/01-mofang-social-avatar.png`：所有真实生成的默认角色身份参考；锁定头型、眼镜、眼神、领结、触手和系统魔方。
- `assets/avatar/02-mofang-worldview-ip-sheet.png`：只用于世界观与完整能力设定图，不用于普通头像。
- `assets/examples/`：只用于校准手绘线条密度、留白、克制色彩和隐喻节奏；除非用户明确要求，否则禁止复刻具体构图。

## 输出口径

- 策略任务：只有用户明确不要生图或在 preflight 中选择文本产物时，才先给 shot list 或 brief，保持短而具体。
- 真实生成：先展示结果；正式交付同时给绝对路径、最终 prompt、风格、参考图角色和 Image Gen 模式。
- 多图任务：逐张列出主题、状态与路径，报告执行策略，明确哪些是最终稿、哪些是被淘汰的迭代稿或失败资产。
- 不把位图描述成矢量稿，不承诺 Illustrator、转曲、印刷或 PDF 生产能力。
