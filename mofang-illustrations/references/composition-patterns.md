# 构图模式与原创规则

每张图只选择一种基础结构，不要混入过多模式。

## 基础结构

- **系统总览**：墨方围绕系统魔方、节点环或 3–6 个模块搭建系统。
- **杠杆隐喻**：墨方校准支点、旋钮或放大装置，让小输入触发大输出。
- **信号与噪音**：墨方观察、过滤或标记少数长期信号。
- **多智能体编排**：墨方连接少量 agent 并将其汇入统一目标，避免组织架构图。
- **反馈闭环**：墨方在工作台把行动、反馈、洞察和重复连接成轻量循环。
- **从混乱到结构**：墨方把云团、线团或碎块整理成规则、流程或系统模块。
- **连接正确节点**：墨方连接少量 people、agents、products、data 或 resources，避免正式拓扑图。
- **学习复利**：墨方通过阅读、记录、修复和回流，让知识逐步形成结构。

## 原创隐喻生成法

1. 把抽象概念变成物理动作：校准、拼接、过滤、压缩、接线、复盘、放大、折叠、排序、沉淀或回流。
2. 把系统结构变成低科技物件：魔方、旋钮、望远镜、白板、工作台、积木、线团、工具箱、旧机器、星图或复盘本。
3. 让墨方亲自承担动作，而不是站在动作旁边。

## 反复刻规则

不要默认打开或复刻 `assets/examples/`。这些图片只校准角色、线条密度、留白、颜色和隐喻节奏。

除非用户明确要求，禁止直接复用：

- 望远镜与围观人群的左右对比。
- 跷跷板撬巨大魔方。
- people/agents/products/data/resources/partners 环形网络。
- agent 六宫格编排。
- 完整 reflect/feedback/insight/action 循环。
- 云团直接输出 framework/rules/process/system。
- chaos → patterns → structure → order 阶段图。

同类主题必须更换主物件、墨方动作或空间关系。
