# 工作流

## 1. 消化内容

提炼核心观点、认知转折、值得配图的段落和只适合文字表达的部分。不要平均配图；优先选择系统边界、信号与噪音、杠杆、反馈、多 agent、从混乱到结构和长期主义判断。

## 2. 选择执行路线

- 只要策略：输出 shot list 或 brief，停止在文本阶段。
- 要真实图片：建立 shot list，逐张调用内置 `image_gen`。
- 要修改现有图片：检查编辑目标，锁定不变项后调用内置 `image_gen` 编辑。

## 3. 建立 shot list

常规内容输出 4–8 张，短内容输出 1–3 张。每张包含：

- 放置位置。
- 核心意思。
- 构图模式。
- 墨方的核心动作。
- 保留项、可改项和禁止项。
- 画幅比例。
- 是否需要真实生成。

## 4. 生成

1. 将社媒头像作为角色身份参考。
2. 按 prompt 模板整理单图需求。
3. 默认无文字、16:9、浅底手绘风。
4. 一张图调用一次 `image_gen`；完成并检查后再生成下一张。
5. 不因“批量”切换 CLI，不使用外部后端或自定义 API 脚本。

## 5. 评图与迭代

按 QA Checklist 检查角色、动作、结构、原创性、比例、留白、色彩和文字。每轮只修一个主要问题，例如“墨方更主动”“减少节点”“移除标题”或“换掉旧案例空间关系”。

## 6. 交付

- 预览：内联展示即可。
- 正式交付：保存最终图、版本化迭代图、shot list 和最终 prompt。
- 回复用户时先给成品与绝对路径，再简述生成与筛选结果。
