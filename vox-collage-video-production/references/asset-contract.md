# 镜头与素材设计合同

## 每场先写镜头卡

在请求任何正式图片前，定义：

1. `knowledge_change`：观众在此场前后多理解了什么。
2. `visual_proposition`：一个可见的因果、比较、组装、揭示、检查、变形或解决机制；不是一串名词。
3. `focal_order`：主焦点、次焦点、最后才可读的辅助信息。
4. `subject_scale`、`facing`、`common_ground`、`occlusion_order` 与 `subtitle_safe_lane`。
5. `beats`：真实口播词、目标帧、对应的视觉动作和声音落点。

每次换镜头即是一份新构图。人物、手、机器和道具不能因为“同一套素材”就被无条件复用；先检查比例、朝向、景别与叙事职责。

## 素材类型

| `asset_type` | 职责 | 例子 |
| --- | --- | --- |
| `core` | 当前镜头的主要信息与焦点 | 人物、机器、书、卷轴、主照片、主卡片 |
| `interaction` | 触发动作或因果 | 手、印章、机械臂、放大镜、夹子 |
| `carrier` | 放置、容纳或组织信息 | 编辑台、轨道、相框、抽屉、打印槽 |
| `support` | 补足节奏、关系和空间 | 色带、纸条、胶带、图钉、胶片、波形条 |
| `foreground` | 遮挡、转场、景深层次 | 撕纸边、前景卡、纸框、遮罩纸片 |
| `background` | 提供稳定环境 | 纸面、网格、档案纹理、半调底图 |
| `code` | 必须精确、由 Remotion 绘制 | 中文、数字、字幕、流程线、图表、波形读数 |

每项资产至少包含：`asset_id`、`asset_type`、`scene_ids`、`purpose`、`reusable`、`transparency_required`、`facing`、`render_path`、`prompt_path`、`status`。进入动画后还要记录最终边界、层级、触发帧、动作和允许遮挡对象。

## 构图与遮挡

- 默认同屏仅一个 `primary` 焦点和最多两个 `secondary` 焦点；背景和字幕不计入。
- 所有未声明的矩形构图区都应分离。素材相碰、被裁掉、挤在边缘或压住后续入场区域，均视为设计错误。
- 仅当遮挡表达明确因果时才允许，例如手拿纸、放大镜检查书页、印章压下卡片。用 `intentional_overlap_with` 和原因明确登记。
- 先安排主视觉的最终位置，再预留手部、纸片、卡片和字幕的运动通道。入口不必放全量元素；将尚未被口播提及的对象留在画外。
- 排比句中每个词组对应独立的入场、揭示或替换；不要在句首一次性展示全部对象。

## 反 PPT 检查

拒绝下列画面：中心标题卡、左文右图模板、整张海报缩放假装动画、字幕是唯一变化层、标签和连线用来挽救弱构图、生成图中承载关键中文或数字、连续场景没有物件/色块/纸带/动作连续性。

安静画面与硬切可以成立，但必须是有意的阅读停留或叙事重置。
